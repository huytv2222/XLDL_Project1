# XLDL_Project1
Đây là báo cáo dự án 1 của ngành Xử Lý Dữ Liệu, tập trung vào việc thực hiện toàn bộ quy trình phân tích và trực quan hóa dữ liệu.
Dự án được thực hiện qua các giai đoạn chính sau:

1. Giới thiệu dự án
1.1 Giới thiệu

Trình bày hiện trạng doanh nghiệp/đơn vị.

Giới thiệu bộ dữ liệu: nguồn dữ liệu, quy mô, đặc điểm.

1.2 Yêu cầu của công ty

Yêu cầu về dữ liệu (cần phân tích gì?).

Yêu cầu về quản lý – lưu trữ dữ liệu.

Mục tiêu cuối cùng của dự án.

Tự đánh giá tính khả thi:

Các kỹ năng hiện có.

Các kỹ năng cần bổ sung.

1.3 Lập kế hoạch dự án

Lập bảng kế hoạch theo mẫu gồm: hạng mục – thời gian – kết quả.

2. Phân tích yêu cầu khách hàng
2.1 Phân tích yêu cầu

Đọc dữ liệu – hiểu cấu trúc – xác định các vấn đề.

Đánh giá yêu cầu doanh nghiệp.

Quyết định công nghệ sử dụng (Power BI, Excel, Database…).

Giải thích lý do chọn công nghệ.

2.2 Xây dựng câu chuyện dữ liệu (Data Storytelling)

Gồm 6 bước:

Đặt vấn đề: Thực trạng, dữ liệu liên quan, mục tiêu.

Xác định câu chuyện: Câu hỏi chính, mục tiêu phân tích, hướng tiếp cận.

Xác định đối tượng: Ai xem báo cáo? Họ biết gì?

Xây dựng câu chuyện chi tiết: Bối cảnh – logic trình bày.

Trình bày dữ liệu: Dùng biểu đồ gì – cho mục đích nào?

Lưu ý quan trọng trong kể chuyện dữ liệu.

2.3 Kiến trúc hệ thống

Vẽ sơ đồ kiến trúc xử lý dữ liệu.

Giải thích luồng dữ liệu từ nguồn → xử lý → lưu trữ → trực quan hóa.

2.4 Giải thích bộ dữ liệu

Giải thích khái niệm nghiệp vụ.

Mô tả các trường dữ liệu, kiểu dữ liệu, ý nghĩa.

3. Làm sạch và chuyển đổi dữ liệu
3.1 Chuẩn bị dữ liệu

Đánh giá giải pháp lưu trữ: Cloud hay On-premise.

Phân bố dữ liệu: replication, phân mảnh, cách phân bố trong phạm vi dự án.

3.2 Làm sạch dữ liệu

Các vấn đề ảnh hưởng chất lượng dữ liệu (null, trùng, sai format…).

Tiêu chí đánh giá chất lượng dữ liệu.

Trình bày từng bước làm sạch (ghi rõ thao tác trên từng cột/dòng).

3.3 Chuyển đổi dữ liệu

Các trường hợp cần chuyển đổi.

Các kỹ thuật (normalize, string transform, date transform, merge…).

Trình bày chi tiết thao tác chuyển đổi trong dự án.

4. Xử lý dữ liệu
4.1 Chuẩn hóa dữ liệu (Power Query)

Chỉnh sửa bảng, đổi tên, đổi kiểu dữ liệu, tách – gộp cột, loại bỏ lỗi.

4.2 Mô hình hóa dữ liệu

Trình bày mô hình (Star Schema, Snowflake...).

Đánh giá mô hình theo tiêu chí chuẩn.

Trình bày chi tiết cách tạo relationships, bảng dimension – fact.

4.3 Xử lý dữ liệu bằng DAX

Measure (Calendar, tính tổng, trung bình, tăng trưởng...).

Calculated Column (phân loại, nhóm dữ liệu…).

Filter (lọc theo ngày, loại, danh mục…).

5. Trực quan hóa dữ liệu
5.1 Các kỹ thuật trực quan hóa

Chọn biểu đồ phù hợp cho từng loại dữ liệu.

5.2 Các nguyên tắc trực quan hóa

Tối giản – đúng mục đích – nhấn mạnh insight – dùng màu hiệu quả.

5.3 Thêm visual mới

Trình bày cách thêm custom visual từ marketplace.

5.4 Tạo các report

Gồm:

Trang thống kê chi tiết

Trang thống kê tổng thể

Mỗi trang cần mô tả:

Bộ lọc

Biểu đồ

Ý nghĩa của biểu đồ

6. Xây dựng báo cáo
6.1 Dashboard & Report

Giải thích sự khác nhau giữa Dashboard và Report.

Lý do cần cả hai trong dự án.

6.2 Xây dựng Dashboard – Report

Mô tả loại dashboard và report bạn tạo.

Giải thích mục đích từng loại.

Bookmark: tạo – mục đích – loại bookmark sử dụng.

Tổng hợp nội dung quan trọng từ Data Storytelling.
